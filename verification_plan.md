# 单词本应用全功能验证流程 (Bug Fix Verification Plan)

这份验证流程将你提到的二十个修复点进行了逻辑分类，串联成了**六个测试场景**。你可以按照这个流程一步步走一遍，就可以覆盖所有已经修复的 bug。

## 场景一：导入与词书管理测试 (Import & Workspace)

**测试目的**：验证批量导入中断、重新选择、跨语言提醒、一键清空及词性解析。

1. **跨语种限制测试**
   - 进入应用的**日语学习模式**。
   - 尝试导入一本**英语**词书（如 CSV 或 JSON），观察系统 **[是否弹出语种不匹配的警告/阻止导入]**【对应 Bug 20】。
2. **导入中断与重置测试**
   - 回到匹配的语种模式，准备导入一个包含多个单词的较大词库。
   - 点击批量导入，在开始上传和解析的过程中，**点击右上角的叉(X)退出**。
   - 再次点击批量导入并上传，观察界面 **[是否发生任务重叠卡死或报错]**【对应 Bug 2】。
   - 再次重新导入，等待 AI 解析完成，此时不点击“确认导入”，而是点击“重新选择文件”。之后再次选择新文件导入，观察 **[是否能正常导入且没有前一次缓存的 bug]**【对应 Bug 2】。
3. **彻底清空测试**
   - 将上述单词成功导入后，尝试清理工作区。
   - 使用右上角选项/批量操作栏中的一键清空功能清空导入的单词。观察 **[是否能一次性清理所有单词，突破了原本20个的限制]**【对应 Bug 19】。
4. **词性分类测试**
   - 重新导入一份包含不同词性（名词、动词、形容词等）的词库。
   - 到达主页后，点击根据词性划分/筛选单词，观察列表 **[是否能正确筛选和显示对应词性]**【对应 Bug 1】。

## 场景二：每日额度与动态调整测试 (Daily Quota)

**测试目的**：验证学习计划中途更改、每日额度超限后的追加功能。

1. **中途修改上限测试**
   - 设置每日新词上限为 **20个**，进入学习（背几个单词后退出）。
   - 前往“设置”，将每日新词上限修改为 **50个**。
   - 回到主页或生词列表，观察整个待学列表/提醒 **[是否随之动态增加到了 50 个的标准]**【对应 Bug 13】。
2. **超额追加测试**
   - 继续背单词，直到完成当日所有的“新词”额度，系统提示今日新词已达上限。
   - 寻找新增的“今日想再多背多少个新词”追加选项卡，选择追加 10 个词。观察 **[是否能成功追加且开始学习进程]**【对应 Bug 4】。

## 场景三：学习过程交互测试 (In-Session Study)

**测试目的**：验证回退、撤销、中断记忆、同音词混淆以及极端 UI 操作。

1. **同音词防串台测试** (特定于日语)
   - 在日语模式下，准备两个读音完全相同但汉字和意思不同的单词（如：`橋 (はし)` 和 `箸 (はし)`）。
   - 进行翻卡复习，观察它们的意思 **[是否独立显示，不再发生混淆或合并]**【对应 Bug 10】。
2. **交互防卡死与回退测试**
   - 在背单词界面，先正常翻过（如点击认识/不认识）一个单词。
   - 然后点击“上一个/回退（撤销）”按钮。
   - 观察当前的选项按钮是否正常渲染，并**尝试敲击空格键、使用鼠标点击**，观察 **[控制是否恢复正常，没有假死]**【对应 Bug 3、Bug 9】。
3. **中途退出重来测试**
   - 学习过程中，将 3 个单词标记为“重来 (Review Again)”。
   - 不要背完这一轮，直接点击**返回控制台 (Dashboard)** 退出。
   - 重新点击“开始学习”，观察你的学习队列 **[是否还保留着那 3 个被标记为重来的单词]**，而不是把它们弄丢了【对应 Bug 8、Bug 16】。

## 场景四：学习完成与冷却池提示 (Cooldown Display)

**测试目的**：验证 FSRS 算法显示、冷却池准确度与预计时间。

1. **倒计时与冷却详情**
   - 背完一轮后，进入“记忆冷却中”的完成界面。
   - 观察页面上 **[是否清晰提示了下一个单词将在几分钟/几小时后可以背]**【对应 Bug 5】。
   - 观察当前立刻可背数量展示，看其 **[是否明确显示当前可背数量？点击刷新时有没有准确更新？]**【对应 Bug 11】。
2. **今日预计时间**
   - 观察完成界面或提示栏，是否提供了一个基于当天的**总预计背诵时间 (Total Estimated Time)**，告诉你今天大约还需要花多长时间才能全部搞定【对应 Bug 12】。
   - 对于预计时间不准确的情况（例如8个单词预计两分钟，但有2分钟后的词），观察新逻辑下 **[预计时间是否计算了需要等待流逝的真实时间(Wall-clock time)]**【对应 Bug 15】。
3. **多尺度冷却显示**
   - 在主页或冷却池列表，查看是否出现了长周期的等待词汇（比如 2小时、8小时后的复习词）。验证冷却池 **[不再仅仅只截取展示 10 分钟内的词，而是清晰暴露了所有待复习区间]**【对应 Bug 6】。

## 场景五：“再练一次”综合机制 (Review Again Rules)

**测试目的**：验证全量数据获取、多端同步与跨语种隔离。

1. **全集覆盖测试**
   - 确保今天你总共复习或新学了比如 30 个单词（分好几个轮次，最后一轮可能只有 2 个单词）。
   - 点击“再练一次”，观察列表 **[是否包含了你今天背过的所有 30 个单词，而不是只有最后一轮的 2 个]**【对应 Bug 7、Bug 14】。
2. **多语言词库隔离**
   - 你在自己的账号里同时学了英语和日语，今天都背了一些。
   - 在某一个语种的工作区下点击“再练一次”，观察 **[是否只出现当前语种的单词，没有把英语和日语混在一起]**【对应 Bug 17】。
3. **多端状态同步测试**
   - 在**电脑端**背完词并点击进入“再练一次”页面，确认工作正常。
   - 保持账号体系，用**手机端** (如果支持网页端响应式或特定App) 登录同一个账号，点击“再练一次”。
   - 观察手机端 **[能否拉取到 PC 端的复习记录，而不再显示“没有可复习记录”]**【对应 Bug 18】。

---

## 如何判定测试通过？

1. **核心流程不阻断**：导入文件和背单词途中的任何退出、重进都不应该引发白屏或空数据队列。
2. **状态机强一致性**：包括“剩余可背单词”、“冷却时间倒数”、“今日已背历史”、“再练一次的集合”，这些数据必须是同一套真实时间的闭环映射。
3. **视觉反馈**：你在每一步都能看到及时的 Toast 提示（如跨语种阻止的红框）或冷却池明确的占位图与倒计时。
